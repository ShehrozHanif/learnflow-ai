# LearnFlow Hackathon 3 Constitution

## Core Principles

### I. Skills-First Development
The Skills are the product. ALL application code must be generated by AI agents (Claude Code and Goose) using Skills. No manual application code writing. Skills must follow the MCP Code Execution pattern: SKILL.md (~100 tokens) + scripts/ (executed, not loaded) + references/ (loaded on-demand).

### II. Token Efficiency (MCP Code Execution Pattern)
Scripts execute outside AI context window. Only minimal results enter context (e.g., "Done!"). Wrap MCP calls in scripts for 77-98% token reduction. SKILL.md must stay under ~100 tokens. Never load full data into context when a script can filter it first.

### III. Cross-Agent Compatibility
All Skills must work on BOTH Claude Code AND Goose without modification. Use the `.claude/skills/` directory structure which both agents read. No agent-specific code in Skills.

### IV. Spec-Driven Development (SDD)
Every phase follows the SpecifyPlus cascade: Constitution -> Specify -> Clarify -> Plan -> Tasks -> Analyze -> Checklist -> Implement -> Ship. No building without specs. Every Skill traces back to: Requirement -> Spec -> Plan -> Task -> Skill -> AI Execution.

### V. Automated Infrastructure
All deployments must be automated via Skills and scripts. Infrastructure lives on Kubernetes (Minikube local, cloud for production). Use Helm charts for stateful services (Kafka, PostgreSQL). All pods must reach Running state before proceeding.

### VI. No Manual Code
DO NOT write application code manually. DO NOT write Skills directly without going through SDD. Let Claude Code and Goose build the actual app using Skills. The developer's job is to write specs and Skills, not application code.

## Quality Standards

### Skills Quality
- Every Skill must have: SKILL.md + scripts/ + references/
- SKILL.md under 500 lines, ideally ~100 tokens for frontmatter + instructions
- Scripts must print minimal output (< 5 lines to context)
- All scripts must be executable and tested
- Skills must include validation/verification steps

### Architecture Standards
- Microservices: FastAPI + Dapr sidecars
- Messaging: Kafka pub/sub for async communication
- Database: Neon PostgreSQL
- API Gateway: Kong with JWT auth
- Auth: Better Auth framework
- Frontend: Next.js + Monaco editor

### Commit Standards
- Commit messages must use agent prefix: "Claude: ..." or "Goose: ..."
- No commits without user approval
- Each phase gets a summary file in `phases/`

## Constraints
- Development in WSL on Windows
- Minikube for local K8s (4 CPU, 8GB RAM)
- Adopt AAIF (Agentic AI Foundation) Standards
- Use context7-efficient for real-time library documentation
- Use skill-creator-pro to generate new skills

## Governance
- AGENTS.md is the single source of truth for project roadmap
- Constitution supersedes all other practices
- Phase completion requires user approval before commit
- PHRs created automatically for every interaction
- ADRs suggested on-demand for significant decisions
